# CLAUDE.md — 다시봄 (Dasibom)

## 정체성
혼자 쓰는 iOS 일정/할 일 앱. **"빨리 적고, 안 까먹게 다시 띄워주는"** 놓침 방지 앱.
무거운 캘린더가 아니라 *초간단 입력 + 능동 재노출*이 본질.

## 🟦 성역 (절대 불변)
- **로컬 우선**: 사용자 데이터는 기기에만 저장, 외부 전송 없음(개인용·프라이버시).
- **빠른 입력 1초 원칙**: 입력 경로는 어떤 변경에서도 더 무거워지지 않는다.
- **놓침 방지가 목적**: 기능 추가가 "적고→다시 보는" 핵심 흐름을 가리면 안 된다.
- 데이터 모델 불변식: 할 일은 `isDone`으로 완료 처리(삭제 아님), `dueAt`/`notifyAt`은 옵셔널.

## 스택 & 컨벤션
- SwiftUI + SwiftData(로컬) + UserNotifications. 백엔드 없음. 테스트 XCTest.
- 단일 사용자. 한국어 UI. 상세 규칙은 `docs/planning/07-coding-convention.md`.

## MVP 범위 (요약 — 상세 `docs/planning/01-prd.md`)
- **Must:** 빠른 한 줄 입력 · 오늘·예정 한눈 보기 · 제때 알림 · 완료 체크
- **Could(나중):** 반복 일정 · 태그 · 외부 캘린더 연동 · 위젯
- **Won't(이번 제외):** 풀 월간 캘린더 뷰 · 공유·협업

## 작업 방식
- 발굴식 바이브 코딩 + LOOP을 따른다. 모호하면 구현 전 질문. 🟨추상 제안은 승인받기.
- **한 번에 하나**(한 변경 = 한 관심사), 완성 기준 먼저, 출처 추적(왜 이렇게 했는지 설명 가능한 코드만).

## 🚨 STOP & ASK 경계선 (자율 진행 금지 — 멈추고 물어볼 것)
- SwiftData 스키마(@Model) 변경 / 데이터 마이그레이션
- 알림(UserNotifications) 권한·정책, 데이터 외부 전송(=🟦성역 침범)
- Won't 항목을 슬쩍 구현(스코프 확대)
- 테스트는 통과하나 "빠른 입력/재노출" 설계 의도가 달라지는 변경
→ 자동 처리 OK: 린트·타입·단순 버그·테스트 추가·문서·네이밍.
