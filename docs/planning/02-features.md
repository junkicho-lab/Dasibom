# 기능 명세 (Features)
> 다시봄 기획 — 기능 명세서

## 1. 우선순위 개요

| 우선순위 | 기능 | 한 줄 정의 |
|---------|------|-----------|
| Must | ⚡ 빠른 한 줄 입력 | 자연어로 1초 안에 할 일을 적고, 날짜/시간 자동 인식 |
| Must | 📋 오늘·예정 한눈 보기 | 오늘 할 일과 다가오는 일정을 한 화면에서 확인 |
| Must | 🔔 제때 알림 | UserNotifications로 정해진 시각에 다시 떠올려 줌 |
| Must | ✅ 완료 체크 | 한 번 탭으로 완료 처리 |
| Could | 🔁 반복 일정 | 매일/매주 등 주기적 반복 (이번 제외, 나중) |
| Could | 🏷️ 태그·카테고리 | 일/일상 등 분류 (이번 제외, 나중) |
| Could | 📅 외부 캘린더 연동 | 시스템 캘린더와 동기화 (이번 제외, 나중) |
| Could | 🧩 홈 위젯 | 오늘 할 일 위젯 (이번 제외, 나중) |
| Won't | 🗓️ 풀 월간 캘린더 뷰 | 이번 버전에서 만들지 않음 |
| Won't | 👥 공유·협업 | 개인용이므로 만들지 않음 |

## 2. Must 기능 상세

### ⚡ Must-1. 빠른 한 줄 입력

- **설명**: 메인 화면 상단 입력바에 자연어 한 줄을 적으면 할 일이 저장된다. "내일 3시 치과"처럼 적으면 날짜(`dueAt`)와 알림 시각(`notifyAt`)을 자동 인식한다. 핵심 가치 "입력 1초 원칙"의 실현 지점.
- **동작**:
  1. 사용자가 입력바에 텍스트 입력 후 완료(엔터/저장).
  2. 자연어 파서가 날짜·시간 표현을 추출해 `dueAt`을 채운다.
  3. 날짜/시간이 인식되면 설정 기본값을 반영한 `notifyAt`을 자동 설정.
  4. 날짜 표현이 없으면 `dueAt = nil`(날짜 미정 항목)로 저장.
  5. 저장 즉시 입력바를 비우고 리스트 상단에 반영.
- **빈 상태**: 입력바에 placeholder "할 일을 적어보세요 (예: 내일 3시 치과)" 노출.
- **실패 상태**:
  - 빈 문자열 저장 시도 → 저장하지 않고 입력바 유지.
  - 날짜 인식 실패 → 날짜 없이 제목만 저장, "날짜를 인식하지 못했어요. 직접 지정할 수 있어요" 안내.
- **수용 기준**:
  - [ ] "내일 3시 치과" 입력 시 제목="치과", `dueAt`=내일 15:00로 저장된다.
  - [ ] 날짜 표현이 없으면 제목만으로 저장되고 `dueAt`은 nil이다.
  - [ ] 빈 입력은 저장되지 않는다.
  - [ ] 저장 후 입력바가 비워지고 즉시 리스트에 나타난다.
  - [ ] 입력→저장 흐름이 추가 화면 이동 없이 1초 내 완료된다.

### 📋 Must-2. 오늘·예정 한눈 보기

- **설명**: 메인 화면에서 오늘 할 일과 앞으로의 일정을 섹션으로 나눠 보여준다. "적어둔 걸 다시 안 봐서 놓친다"는 문제를 화면 차원에서 해결.
- **동작**:
  1. `dueAt` 기준으로 "오늘", "예정"(미래), "날짜 없음"으로 구분 정렬.
  2. 오늘 섹션을 최상단에 노출, 시간 빠른 순 정렬.
  3. 완료된 항목은 흐리게 처리하거나 하단으로 내림.
- **빈 상태**: "오늘 할 일이 없어요. 위에서 빠르게 적어보세요" 안내 + 입력바 강조.
- **실패 상태**: 데이터 로드 실패 시 "목록을 불러오지 못했어요. 다시 시도" 재시도 버튼.
- **수용 기준**:
  - [ ] 오늘 마감 항목이 "오늘" 섹션에 표시된다.
  - [ ] 미래 항목이 "예정" 섹션에 날짜순으로 표시된다.
  - [ ] 날짜 없는 항목이 별도 섹션에 표시된다.
  - [ ] 항목이 하나도 없으면 빈 상태 안내가 보인다.

### 🔔 Must-3. 제때 알림

- **설명**: `notifyAt` 시각에 로컬 알림을 보내 사용자가 다시 떠올리게 한다. 능동적 재노출의 핵심.
- **동작**:
  1. 항목 저장/수정 시 `notifyAt`이 있으면 `UNUserNotificationCenter`에 알림 예약.
  2. 항목 삭제/완료 시 예약 알림 취소.
  3. 최초 알림 권한 미허용 시 권한 요청.
- **빈 상태**: `notifyAt`이 없는 항목은 알림을 예약하지 않는다(정상).
- **실패 상태**:
  - 알림 권한 거부 → 설정 화면 안내 배너 "알림이 꺼져 있어요. 설정에서 켜주세요".
  - 과거 시각 예약 시도 → 예약하지 않고 무시.
- **수용 기준**:
  - [ ] `notifyAt`이 있는 항목은 해당 시각에 로컬 알림이 온다.
  - [ ] 완료/삭제한 항목의 예약 알림은 취소된다.
  - [ ] 권한이 없으면 권한 요청 또는 안내 배너가 노출된다.
  - [ ] 과거 시각은 알림을 예약하지 않는다.

### ✅ Must-4. 완료 체크

- **설명**: 항목을 한 번의 탭으로 완료 처리(`isDone = true`)한다.
- **동작**:
  1. 리스트 항목의 체크 영역 탭 → `isDone` 토글.
  2. 완료 시 예약 알림 취소, 시각적으로 완료 표시.
  3. 다시 탭하면 미완료로 복귀(알림 시각이 미래면 재예약).
- **빈 상태**: 해당 없음(항목이 있을 때만 동작).
- **실패 상태**: 저장 실패 시 토글 롤백 + "변경을 저장하지 못했어요" 안내.
- **수용 기준**:
  - [ ] 체크 탭 시 `isDone`이 true로 바뀐다.
  - [ ] 완료 항목의 예약 알림이 취소된다.
  - [ ] 다시 탭하면 미완료로 복귀한다.
  - [ ] 완료 항목이 시각적으로 구분된다.

## 3. Could (나중에 검토)

| 기능 | 비고 |
|------|------|
| 🔁 반복 일정 | 스키마 확장 시 `recurrenceRule` 필드 추가 여지 |
| 🏷️ 태그·카테고리 | 스키마 확장 시 `tags` 필드 추가 여지 |
| 📅 외부 캘린더 연동 | EventKit 연동 검토 (로컬 우선 성역 준수 전제) |
| 🧩 홈 위젯 | WidgetKit, 오늘 섹션 재사용 |

> Could 기능은 이번 스키마에 필드로 포함하지 않는다. 확장 여지만 기록.

## 4. Won't (이번 버전 명시적 제외)

| 기능 | 제외 이유 |
|------|----------|
| 🗓️ 풀 월간 캘린더 뷰 | 핵심 가치(초간단 입력+재노출)와 무관, 복잡도 증가 |
| 👥 공유·협업 | 단일 사용자 개인용 앱 |
