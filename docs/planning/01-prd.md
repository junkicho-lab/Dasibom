# 01 · PRD — 다시봄 (Dasibom)

> 혼자 쓰는 iOS 놓침 방지 앱. "빨리 적고, 안 까먹게 다시 띄워준다."

## Problem Reframing (문제 재정의) — 핵심

| 구분 | 내용 |
|---|---|
| **처음 문제** | "일정관리 앱이 필요하다" |
| **재정의** | 진짜 문제는 *알림 부재*가 아니라 → ① **입력이 무거워서 안 적고**, ② **적어둔 걸 다시 안 봐서** 놓친다 |
| **바뀐 이유** | 사용자가 "놓치는 장면"으로 *입력 귀찮음 + 재노출 실패*를 지목. 캘린더 기능 부족이 아님 |
| **핵심 가치** | **초간단 입력 + 능동 재노출** (무거운 캘린더 아님) |

## 사용자
- 나 혼자(단일 사용자), 일·일상이 섞임. 로컬 우선·프라이버시 중시.

## MVP Scope

| 등급 | 항목 |
|---|---|
| **Must (이번 출시)** | ⚡빠른 한 줄 입력(날짜/시간 자연어 인식) · 📋오늘·예정 한눈 보기 · 🔔제때 알림 · ✅완료 체크 |
| **Should** | 빠른 입력 시 날짜 미지정 항목의 "언제든" 처리, 간단 정렬(시간순) |
| **Could (나중)** | 반복 일정 · 태그·카테고리 · 외부 캘린더 연동 · 홈 위젯 |
| **Won't (이번 제외)** | 🗓️풀 월간 캘린더 뷰 · 👥공유·협업 |

> **Won't이 MVP의 핵심.** 월간 캘린더·공유를 빼야 "놓침 방지"에 집중한 출시 가능한 첫 버전이 된다.

## Must별 인수 기준 (Acceptance Criteria)
- **빠른 한 줄 입력**: 앱 진입 후 3탭 이내 저장. "내일 3시 치과" 입력 시 제목="치과", dueAt=내일 15:00 자동 파싱.
- **오늘·예정 한눈 보기**: 메인 진입 시 오늘 항목 + 다가오는 항목이 시간순으로 보인다(빈 상태 안내 포함).
- **제때 알림**: notifyAt 설정 시 해당 시각에 로컬 알림 발송, 탭하면 해당 항목으로 이동.
- **완료 체크**: 항목 체크 시 완료 처리(삭제 아님), 오늘 목록에서 정리됨.

## 성공 기준 (출시 판단)
- 핵심 흐름(입력→오늘 노출→알림→완료)이 끊김 없이 동작.
- 빌드·테스트·타입·린트 통과(상세는 LOOP.template / verification).
- 데이터는 전부 로컬(외부 전송 0).

## 비범위·리스크
- 타임존/DST 경계 날짜 계산 — 테스트 필요.
- 자연어 날짜 파싱 정확도 — 실패 시 사용자가 직접 시간 지정 가능해야 함.
