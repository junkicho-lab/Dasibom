# 기술 스택 (Tech Stack)
> 다시봄 기획 — 기술 스택 결정

## 1. 스택 결정 요약

| 영역 | 선택 | 트레이드오프 (한 줄) |
|------|------|---------------------|
| UI | SwiftUI | 선언형으로 빠른 개발·적은 코드 ↔ 일부 고급 커스터마이징은 UIKit보다 제약 |
| 영속성 | SwiftData | 모던·SwiftUI 친화적·보일러플레이트 적음 ↔ CoreData보다 신생이라 사례·세밀 제어 부족 |
| 알림 | UserNotifications | OS 표준 로컬 알림으로 백엔드 불필요 ↔ 정확 시각 보장은 OS 정책에 종속 |
| 자연어 파싱 | Foundation(NSDataDetector/Date) | 의존성 없이 날짜 인식 ↔ 복잡한 한국어 표현은 자체 규칙 보강 필요 |
| 아키텍처 | 경량 MVVM (@Observable) | 화면 3개에 적정 복잡도 ↔ 대규모 확장 시 구조 재정비 필요 |
| 테스트 | XCTest | Xcode 기본 통합·러닝커브 낮음 ↔ BDD 스타일은 별도 도구 필요 |

## 2. SwiftData vs CoreData

| 기준 | SwiftData | CoreData |
|------|-----------|----------|
| 코드량 | 적음 (`@Model`) | 많음 (xcdatamodeld, NSManagedObject) |
| SwiftUI 통합 | 네이티브 (`@Query`) | 래핑 필요 |
| 성숙도 | 신생 (iOS 17+) | 오래됨, 안정적 |
| 세밀 제어 | 제한적 | 강력 |
| 결론 | **채택** — 단일 사용자·단순 스키마에 최적 | 미채택 — 이 규모엔 과함 |

> 로컬 우선(성역)과 단순 스키마(Task 1종)에는 SwiftData가 명확히 유리.

## 3. 최소 iOS 버전 제안

| 후보 | 근거 |
|------|------|
| **iOS 17.0 (권장)** | SwiftData 최소 지원 버전. 개인용 단일 사용자라 구버전 호환 부담 없음. |
| iOS 18.0 | 최신 API 활용 가능하나 기기 커버리지 축소. 필요 시 상향. |

> 개인용이므로 본인 기기 버전 기준으로 17.0 이상 채택. 백엔드/배포 호환 이슈 없음.

## 4. 핵심 프레임워크

| 프레임워크 | 용도 |
|-----------|------|
| SwiftUI | 화면 3종 구현 |
| SwiftData | Task 영속 저장 |
| UserNotifications | 로컬 알림 예약/취소 |
| Foundation | 날짜 파싱(NSDataDetector), UUID, Date |

## 5. 테스트 도구

| 대상 | 도구 | 비고 |
|------|------|------|
| 단위 테스트 | XCTest | 파서, 스케줄러, ViewModel 분류 로직 |
| UI 테스트 | XCUITest (선택) | 핵심 입력→저장 흐름 |

- 우선 검증: `NaturalLanguageParser`("내일 3시 치과" 파싱), `MainViewModel` 분류/정렬, `NotificationScheduler` 예약/취소.

## 6. 비기능 / 성역 준수

- **로컬 우선**: 네트워크 라이브러리·외부 SDK·분석 도구 미사용.
- **빠른 입력 1초**: 파싱은 동기·경량으로 유지, 입력→저장 지연 최소화.
- **사용자 데이터 소유**: 외부 전송 없음, 기기 내 보관.
