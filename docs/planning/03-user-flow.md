# 사용자 흐름 (User Flow)
> 다시봄 기획 — 핵심 사용자 흐름

## 1. 핵심 흐름 한 줄 요약

> 떠오름 → 1초 입력 → 저장 → 오늘 뷰 노출 → 제때 알림 → 완료 체크

JTBD: "갑자기 할 일이 떠오르면 1초 만에 적고, 나중에 제때 다시 떠올라 안 놓친다."

---

## 2. 단계별 흐름

| 단계 | 사용자 행동 | 시스템 반응 | 관련 데이터 |
|------|-----------|-----------|-----------|
| 1. 떠오름 | 할 일이 생각남 → 앱 실행 | 메인 화면, 입력바에 포커스 가능 | - |
| 2. 입력 | "내일 3시 치과" 한 줄 입력 | 자연어 파싱(날짜/시간 추출) | `title`, `dueAt`, `notifyAt` |
| 3. 저장 | 완료(엔터) | SwiftData 저장, 입력바 비움 | `createdAt` 기록 |
| 4. 노출 | (수동 아님) | 오늘/예정 섹션 분류 후 리스트 반영 | `dueAt` 기준 정렬 |
| 5. 예약 | (자동) | `notifyAt`로 로컬 알림 예약 | `UNUserNotificationCenter` |
| 6. 알림 | 정해진 시각 | 알림 수신 → 탭하면 앱 진입 | `notifyAt` 도달 |
| 7. 완료 | 체크 탭 | `isDone=true`, 예약 알림 취소 | `isDone` |

## 3. 텍스트 플로우 다이어그램

### 메인 흐름 (해피 패스)

```
[할 일 떠오름]
      │
      ▼
[앱 실행 → 메인]
      │
      ▼
[상단 입력바에 "내일 3시 치과" 입력]
      │
      ▼
[엔터 / 저장]
      │
      ▼
[자연어 파싱] ──날짜 인식?──┐
      │ 예                   │ 아니오
      ▼                      ▼
[dueAt + notifyAt 설정]   [제목만 저장 (dueAt=nil)]
      │                      │
      └──────┬───────────────┘
             ▼
      [SwiftData 저장]
             │
             ▼
   [오늘/예정/날짜없음 분류] ──▶ [메인 리스트 노출]
             │
             ▼ (notifyAt 있으면)
      [로컬 알림 예약]
             │
        ┄┄┄ (시간 경과) ┄┄┄
             │
             ▼
      [notifyAt 시각 → 알림 발송]
             │
             ▼
      [사용자 알림 확인 → 앱 진입]
             │
             ▼
      [체크 탭 → isDone=true]
             │
             ▼
      [예약 알림 취소 → 완료 표시]
```

### 분기 흐름

```
[알림 권한 거부]
      │
      ▼
[설정 화면 안내 배너 노출 → "설정에서 켜기"]

[빈 입력 저장 시도]
      │
      ▼
[저장 안 함, 입력바 유지]

[완료 항목 다시 탭]
      │
      ▼
[isDone=false 복귀 → notifyAt 미래면 알림 재예약]
```

## 4. 화면 전환 맵

| 출발 | 동작 | 도착 |
|------|------|------|
| 메인 | 입력바 저장 | 메인(리스트 갱신) |
| 메인 | 항목 탭 | 추가/편집 |
| 메인 | 설정 버튼 | 설정 |
| 추가/편집 | 저장/취소 | 메인 |
| 알림 | 알림 탭 | 메인(해당 항목) |
| 설정 | 뒤로 | 메인 |
