# 10 · Desire Map — 다시봄 (Dasibom)

> 표면 요청 → 진짜 문제 → JTBD → Must↔JTBD 연결 → Won't과 이유.

## 표면 요청
"일정관리 앱 만들어줘"

## 재정의된 핵심 문제
알림이 없어서가 아니라 — **① 입력이 무거워 안 적고, ② 적어둔 걸 다시 안 봐서** 놓친다.
→ 핵심 가치: **초간단 입력 + 능동 재노출**.

## JTBD
> "갑자기 할 일·약속이 떠오르면 → 1초 만에 적고 → 나중에 제때 다시 떠올라 안 놓친다."

전형적 장면: *입력이 귀찮아 안 적거나, 적어둔 걸 다시 안 봄.*

## Must ↔ JTBD 연결

| Must 기능 | JTBD의 어느 부분을 해결하나 |
|---|---|
| ⚡ 빠른 한 줄 입력 | "1초 만에 적고" — 입력 마찰 제거(안 적는 문제) |
| 📋 오늘·예정 한눈 보기 | "다시 떠올라" — 적은 걸 능동 재노출(안 보는 문제) |
| 🔔 제때 알림 | "제때" — 그 시점에 push (때 놓침) |
| ✅ 완료 체크 | 처리한 건 정리 → 재노출 목록을 신뢰 가능하게 유지 |

## Won't과 이유

| Won't | 왜 이번엔 빼나 |
|---|---|
| 🗓️ 풀 월간 캘린더 뷰 | 목적은 *놓침 방지*지 일정 시각화가 아님. 무거워지면 핵심 흐림 |
| 👥 공유·협업 | 개인용. 로컬 우선 🟦성역과 충돌, 범위 폭발 |

## 🟦 성역 (이 욕구를 지키는 불변선)
로컬 우선 · 빠른 입력 1초 · "적고→다시 본다" 흐름 보호.
