# 페르소나 (Persona)
> 다시봄 기획 — 주 페르소나와 시나리오

## 1. 주 페르소나

| 항목 | 내용 |
|------|------|
| 이름 | 김민수 (가명) |
| 나이/상황 | 34세, 직장인. 일과 일상이 뒤섞여 돌아감 |
| 한 줄 정의 | "머릿속에 떠오른 할 일을 적기 귀찮아 미루다 까먹고, 적어둬도 다시 안 봐서 놓치는 사람" |
| 기기 | 아이폰 사용. 알림은 받지만 앱을 잘 열지 않음 |
| 디지털 성향 | 복잡한 생산성 앱은 며칠 쓰다 포기한 경험 다수 |

### 목표 (Goals)
- 떠오른 즉시 부담 없이 적고 싶다.
- 정작 필요한 순간에 잊지 않고 다시 떠올리고 싶다.
- 앱 사용법을 따로 배우고 싶지 않다.

### 불편 (Pain Points)
- 할 일 입력이 번거로워(날짜·카테고리 선택) 결국 안 적는다.
- 메모는 했지만 다시 열어보지 않아 마감을 놓친다.
- 기능 많은 앱은 오히려 정리에 시간을 뺏긴다.

### 기대 가치
- ⚡ 한 줄이면 끝나는 입력.
- 🔔 알아서 제때 다시 알려줌.
- 🔒 내 데이터가 내 기기에만 있다는 안심.

## 2. 시나리오

### "치과 예약을 놓치지 않은 날"

| 시점 | 상황 | 다시봄 사용 |
|------|------|------------|
| 화요일 오후, 통화 중 | 치과에서 "내일 오후 3시 가능하세요?"라고 함. 메모할 손이 없음 | 통화 후 앱 실행 |
| 통화 직후 | 잊기 전에 빠르게 적고 싶음 | 입력바에 "내일 3시 치과" 입력 → 저장. dueAt=수요일 15:00, notifyAt 자동 설정 (1초) |
| 그 사이 | 평소처럼 앱을 다시 열지 않음 | 메인 "예정" 섹션에 항목 남아 있음 |
| 수요일 낮 | 일에 집중하느라 예약을 잊음 | 다시봄이 설정 시각에 알림 발송 → "치과" |
| 알림 확인 | "아, 치과 있었지" | 알림 탭 → 앱 진입, 일정 확인 |
| 진료 후 | 다녀옴 | 항목 체크 탭 → 완료 처리, 예약 알림 정리 |

### 결과
- 적는 마찰이 사라져 **실제로 적었고**, 능동 재노출 덕에 **놓치지 않았다**.
- 김민수의 JTBD("떠오르면 1초에 적고, 제때 다시 떠올라 안 놓친다")가 충족됨.
