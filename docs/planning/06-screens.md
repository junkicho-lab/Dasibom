# 06 · Screens — 다시봄 (Dasibom)

> `/screen-spec`의 입력 문서. 화면 3개로 MVP를 닫는다.

## 화면 목록

| # | 화면 | 목적 | Must 매핑 |
|---|---|---|---|
| S1 | **메인 (오늘·예정)** | 적어둔 걸 다시 보게 하는 핵심 화면 | 오늘·예정 한눈 보기, 빠른 입력, 완료 체크 |
| S2 | **추가/편집** | 항목 생성·수정 | 빠른 한 줄 입력, 알림 시각 설정 |
| S3 | **설정** | 알림 기본값 등 | 제때 알림(기본 설정) |

## S1 · 메인 (오늘·예정)
- **상단 빠른 입력바**: 항상 보이는 한 줄 입력. 텍스트 입력 → 엔터/+ → 즉시 저장(자연어 날짜 파싱). *1초 입력 = 🟦성역.*
- **본문 리스트**: `오늘` 섹션 + `예정`(다가오는) 섹션, 시간순. 각 항목 = 체크박스 + 제목 + 시간.
- **완료 체크**: 항목 좌측 체크 → 완료 처리(삭제 아님, 흐려지거나 정리).
- **빈 상태**: "할 일이 없어요. 위에 한 줄로 적어보세요." + 입력바 강조.
- **항목 탭** → S2(편집).
- **흐름 1줄**: 입력바에 적기 → 오늘/예정에 바로 노출 → (시간 되면) 알림 → 체크.

## S2 · 추가/편집
- 필드: 제목(필수), 날짜·시간(옵셔널, 자연어 파싱 결과 미리 채움), 알림 시각(옵셔널, 기본=마감 시각).
- 액션: 저장 / (편집 시) 삭제.
- **실패 상태**: 날짜 파싱 실패 시 제목만 저장 + "시간은 직접 정해주세요" 안내(입력은 절대 막지 않음).
- 빈 제목 저장 차단.

## S3 · 설정
- 알림 기본 동작(예: 마감 시각에 알림 켜기 on/off), 기본 알림 리드타임(정시/10분 전 등).
- (Could 자리표시) 반복·태그·연동은 비활성/숨김 — 이번 버전 없음.

## 화면 전이도
```
        ┌──────────── S3 설정 ───────────┐
        │            ▲                    │
   [+]  │            │ 설정              │
S1 메인 ─┼── 항목 탭 ─▶ S2 추가/편집 ──저장/삭제──▶ S1
 (입력바·오늘·예정·체크)
        └── 빠른 입력바: S2 안 거치고 바로 저장 ──┘
```

## 데이터 요구 (data_requirements)
- S1: 오늘/예정 항목 목록 = `Item { title, dueAt, isDone, notifyAt }` 조회(미완료 우선, 시간순).
- S2: 단일 `Item` 생성/수정.
- S3: 알림 기본값(앱 설정).
> 화면에 쓰는 필드 = 데이터 모델 필드와 1:1 (sync 드리프트 방지).
