# 경쟁 분석 (Competition)
> 다시봄 기획 — 유사 서비스 비교

## 1. 비교 대상

- **Apple 미리알림 (Reminders)**: iOS 기본 할 일 앱.
- **Todoist**: 크로스플랫폼 생산성 태스크 매니저.
- **네이버 캘린더**: 일정 중심 캘린더 앱.

## 2. 비교표

| 항목 | 다시봄 | Apple 미리알림 | Todoist | 네이버 캘린더 |
|------|--------|---------------|---------|--------------|
| 핵심 성격 | 놓침 방지 (입력+재노출) | 할 일 목록 | 생산성 관리 | 일정 캘린더 |
| 입력 속도 | ⚡ 한 줄 자연어 1초 | 보통 (필드 입력) | 자연어 지원하나 기능 많음 | 폼 입력, 느림 |
| 자연어 날짜 인식 | ✅ ("내일 3시 치과") | 부분 (Siri/일부) | ✅ | ❌ |
| 능동 재노출 | ✅ 알림으로 다시 띄움 | ✅ 알림 | ✅ 알림 | ✅ 알림 |
| 학습 부담 | 매우 낮음 (화면 3개) | 낮음 | 높음 (기능 과다) | 중간 |
| 로컬 우선/프라이버시 | ✅ 외부 전송 없음 | iCloud 동기화 | 클라우드 계정 필수 | 계정·서버 기반 |
| 월간 캘린더 뷰 | ❌ (의도적 제외) | △ | ✅ | ✅ |
| 협업·공유 | ❌ (개인용) | ✅ | ✅ | ✅ |
| 대상 | 개인, 일·일상 혼재 | 일반 | 팀/파워유저 | 일정 관리 사용자 |

## 3. 핵심 통찰

- 경쟁 앱들은 **기능이 많아 입력이 무겁거나**, **적어둔 걸 다시 보게 만드는 동기가 약하다**.
- 진짜 문제는 알림 부재가 아니라 ① 입력이 무거워 안 적고 ② 다시 안 봐서 놓치는 것.

## 4. 차별화 한 줄

> **"다르다가 아니라 낫다"** — 다시봄은 새로운 기능을 더하지 않는다. 대신 *초간단 한 줄 입력*으로 적는 마찰을 없애고, *제때 능동 재노출*로 다시 보게 만들어 "놓침"이라는 단 하나의 문제를 누구보다 잘 해결한다.
