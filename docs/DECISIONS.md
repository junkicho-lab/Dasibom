# DECISIONS — 다시봄 (결정과 이유)

> ADR 스타일. 결정 + *왜*. 변경 시 날짜·이유 기록.

## 2026-06-23 — 기획 결정 (socrates)

- **D1. 문제 재정의: "알림 부재"가 아니라 "입력 마찰 + 재노출 실패".**
  이유: 사용자가 놓치는 장면으로 *입력 귀찮음 + 적어둔 걸 다시 안 봄*을 지목. 캘린더 기능 부족이 아님.
- **D2. 핵심 가치 = 초간단 입력 + 능동 재노출.** 무거운 캘린더 지양.
- **D3. 플랫폼 = iOS 네이티브(SwiftUI).**
  이유: 빠른 입력·제때 알림·위젯·"항상 손에"에 모바일이 유리, iOS 네이티브가 알림/위젯에 강함. 사용자 SwiftUI 경험.
- **D4. 저장 = SwiftData 로컬, 백엔드 없음(로컬 우선).**
  이유: 개인용·프라이버시(🟦성역). 동기화는 범위·복잡도 폭발 → 제외.
- **D5. Won't = 풀 월간 캘린더 뷰 · 공유·협업.**
  이유: 목적이 놓침 방지라 시각화·협업은 핵심을 흐림. 출시 가능성 위해 컷.
- **D6. 데이터 모델 최소: Item{id,title,dueAt?,isDone,createdAt,notifyAt?}.**
  반복/태그 필드는 Could라 *이번 스키마 제외*(확장 여지만). 이유: 스키마 변경은 🚨경계선.
- **D7. 완료는 삭제가 아니라 isDone 처리.** 이유: 재노출 목록 신뢰성·실수 복구.
